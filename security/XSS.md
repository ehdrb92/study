XSS(Cross-Site Scripting)는 웹 애플리케이션에서 자주 발생하는 보안 취약점 중 하나이다. 공격자가 악의적인 스크립트를 웹 페이지에 삽입하여 다른 사용자의 브라우저에서 실행되도록 하는 것이다. 핵심은 사용자가 의도하지 않은 자바스크립트 코드가 브라우저에서 실행되도록 하는 것이다.

## 주요 유형

- Stored XSS
    악성 스크립트가 데이터베이스에 저장된 후 여러 사용자에게 전파됩니다.
    예시: 게시판 글, 댓글, 사용자 프로필 등에 스크립트를 심어둠.
- Reflected XSS
    악성 스크립트가 서버 응답에 바로 반영되어 실행됩니다.
    예시: 검색창에 입력한 값을 그대로 화면에 출력하는 경우, 공격자가 조작된 URL을 피해자에게 클릭하도록 유도.
- DOM-based XSS
    서버가 아닌 클라이언트 측 자바스크립트에서 DOM 조작을 통해 스크립트가 실행되는 경우.
    예시: `location.hash`나 `document.write()` 같은 부분이 제대로 필터링되지 않았을 때 발생.

## 방어 기법

- 출력 시 인코딩: HTML, JS, CSS, URL 각각에 맞는 컨텍스트 인코딩 적용
- 입력 검증: 스크립트 태그나 이벤트 핸들러 같은 의심스러운 입력 필터링
- CSP(Content Security Policy): 허용된 소스만 실행하도록 정책 설정
- HttpOnly 쿠키: 자바스크립트에서 쿠키 접근 불가하도록 설정
- 라이브러리/프레임워크 기능 활용
