데이터베이스에서 참조 관계에 있는 데이터 간에 무결성 보장을 위한 `ON DELETE` 옵션이 존재한다. 해당 옵션은 참조 관계에서 부모에 해당하는 데이터가 삭제될 경우 자식에 해당하는 데이터들을 어떻게 처리할지에 대한 정책을 경정하는 것이다.

각 옵션에 대한 처리는 다음과 같이 진행된다.

- `CASCADE`: 부모 데이터가 삭제되면, 자식 데이터도 함께 삭제된다. 해당 옵션은 잘못 사용할 경우 연쇄 삭제로 문제가 생길 수 있다. 부모 데이터 없이는 아무런 의미가 없는 데이터만 해당 옵션을 반영해야한다.
- `SET NULL`: 부모 데이터가 삭제되면, 참조만 끊고(외래 키 컬럼 값을 null) 자식 데이터를 살려둔다. 로그 혹은 이력 데이터가 필요할 경우 사용한다.
- `RESTRICT`: 자식 데이터가 존재하면, 부모 데이터의 삭제를 차단한다. 일반적으로 데이터베이스에서 ON DELETE 옵션의 기본값이다. 실수로 부모 데이터가 삭제되면 안되거나, 어떤 결정을 내릴지 애매한 상황이라면, 해당 옵션을 사용한다.
