베어 메탈 서버(Bare Metal Server)란 무엇인가? 쿠버네티스와 같은 가상화 관련 정보를 확인하는 중 베어 메탈이라는 용어가 등장하여 알아보게 되었다.

베어 메탈 서버는 물리적 서버 하드웨어에 직접 운영체제를 설치하여 사용하는 서버 환경을 의미한다. 가상화 계층이나 하이퍼바이저 없이 하드웨어 리소스에 직접 접근할 수 있는 형태이다.

베어 메탈 서버라고 불릴 수 있는 예시는 다음과 같다.

- AWS EC2 베어 메탈 인스턴스: i3.metal, m5.metal, c5.metal 등 특정 인스턴스 타입에서 물리 서버를 그대로 제공
- Google Cloud Bare Metal Solution: Oracle 데이터베이스나 SAP 같은 특수 워크로드용으로 물리 서버를 제공
- IBM Cloud Bare Metal Servers: 시간 단위 또는 월 단위로 물리 서버를 임대
- 코로케이션(Colocation): 자신의 하드웨어를 데이터센터에 설치하고 전력, 냉각, 네트워크만 제공받는 서비스
- 전용 서버 임대: 호스팅 업체가 소유한 물리 서버를 단독으로 임대받아 사용하는 형태
- 온프레미스 데이터센터: 기업이 자체적으로 운영하는 물리 서버
- 엣지 컴퓨팅 서버: 지역별로 배치된 물리 서버들로 낮은 레이턴시가 필요한 서비스에 사용

위 예시들 외에도 개인적으로 서버 컴퓨터 머신을 구매하고 운영체제를 설치 후 별도의 가상화 기술 없이 서버를 구축 및 배포하면 이 또한 베어 메탈 서버라고 할 수 있다.

베어 메탈 서버의 주요 특정은 다음과 같다.

- 하드웨어 직접 접근: CPU, 메모리, 스토리지, 네트워크 등 모든 하드웨어 리소스를 가상화 오버헤드 없이 직접 사용할 수 있다.
- 성능 최적화: 가상화 계층이 없어서 최대 성능을 발휘할 수 있으며, 특히 CPU 집약적이거나 I/O 집약적인 작업에서 우수한 성능을 보인다.
- 완전한 제어권: 서버의 모든 설정을 직접 제어할 수 있어 특정 요구사항에 맞는 커스터마이징이 가능하다.
