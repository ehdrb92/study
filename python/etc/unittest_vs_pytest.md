unittest의 특징

- Python 표준 라이브러리이기 때문에 별도의 설치가 필요없다.
- 자바의 JUnit 테스팅 프레임워크의 영향을 크게 받았다.
- unittest.TestCase를 상속받는 클래스 내부에 테스트를 작성하는 방식이기 때문에 클래스 작성이 필수적이다.
- setUp, tearDown 함수로 환경 설정 용이
- 파이썬임에도 카멜 케이스로 작성한다.

pytest의 특징

- 불필요하게 클래스 작성이 필요없고, assert를 이용해 상대적으로 간결한 코드 작성이 가능하다.
- 명시적인 이름을 가지는 픽스쳐를 선언하여 구분이 명확하다.
- 특정 픽스쳐가 다른 픽스쳐를 사용가능하다.
- 병렬 테스트

unittest 언제 사용할까?

- pytest만의 독특한 테스트 방식이 생산성 향상에 이점을 줄 수 없을것 같다.
- JUnit과 같은 다른 테스트 프레임워크 작성 방식에 익숙하다.
- 픽스쳐 주입 혹은 assert문 재작성과 같은 문제를 피하고 싶다.

pytest 언제 사용할까?

- 간결하고 파이썬스러운 코드를 작성하고 싶다.
- 매개변수화된 픽스쳐나 병렬 테스트와 같은 고급 기능이 필요하다.
