## 공백

- 탭 대신 스페이스로 들여쓰기하라
- 들여쓰기에는 4칸 스페이스를 사용하라
- 라인 길이는 79개 문자 이하여야 한다
  - 이 부분은 확인이 필요할듯
- 긴 식을 다음 줄에 이어서 쓸 경우 일반적인 들여쓰기보다 4 스페이스를 더 들여써야 한다.
- 스크립트 내부에서 함수와 클래스 사이에는 빈 줄을 두 줄 넣어라
- 클래스 내부에서 메서드 사이에는 빈 줄을 한 줄 넣어라
- 딕셔너리에서 키와 콜론 사이에는 공백을 넣지 않고, 한 줄 안에 키와 값을 넣으면 콜론 다음에 스페이스를 하나 넣는다
- 변수 대입에서 = 전후에는 스페이스를 하나씩만 넣는다
- 타입 표기를 덧붙이는 경우에는 변수 이름과 콜론 사이에 공백을 넣지 않고, 콜론과 타입 사이에 스페이스를 하나 넣어라

## 명명 규약

- 함수, 변수, 속성은 snake_case를 사용한다
- 보호되야 하는 인스턴스 속성은 _ 로 시작한다
- 비공개(private, 한 클래스 내부에서만 사용되고 다른 곳에서 조회되면 안되는 속성) 속성은 __ 로 시작한다
- 클래스는 PascalCase를 사용한다
- 모듈 수준의 상수는 UPPER_CASE를 사용한다
- 클래스의 인스턴스 메서드는 호출 대상 객체를 가리키는 첫 번째 인자의 이름으로 반드시 self를 사용한다
- 클래스 메서드는 클래스를 가르키는 첫 번째 인자의 이름으로 반드시 cls를 사용한다

## 식과 문

- 긍정적인 식을 부정하지 말고(if not a is b), 부정을 내부에 넣어라(if a is not b)
- 빈 컨테이너나 시퀀스를 검사할 때는 길이를 0과 비교하지 말아라
  - 빈 컨테이너나 시퀀스는 암묵적으로 False로 취급된다는 사실을 통해 "if not {컨테이너}" 조건문을 써라
  - 마찬가지로 비어있지 않은 컨테이너나 시퀀스의 경우에도 "if {컨테이너}" 조건문을 써라
- 한 줄짜리 if 문이나 한 줄 짜리 for, while 반복문, 한 줄짜리 except 복합문을 쓰지마라
  - 명확성을 위해 각 부분을 여러 줄에 나눠 배치하라
- 식을 한 줄안에 다 쓸 수 없는 경우, 식을 괄호로 둘러싸고 줄바꿈과 들여쓰기를 추가해서 읽기 쉽게 하라
- 여러 줄에 걸쳐 식을 쓸 때는 줄이 계속된다는 표시를 하는 \ 문자보다는 괄호를 사용하라

## 불러오기

- import 문을 항상 스크립트 맨 앞에 위치시켜라.
- 모듈을 불러올 때는 절대적인 이름을 사용하고, 현 모듈의 경로에 상대적인 이름은 사용하지 말아라
- 반드시 상대적인 경로로 임포트해야 하는 경우에는 from . import foo처럼 명시적인 구문을 사용하라
- 불러오기를 할 때는 표준 라이브러리, 서드 파티, 본인이 만든 모듈 순서로 섹션을 나눠라
  - 각 섹션은 알파벳 순서로 모듈을 불러와라
