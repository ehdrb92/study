## disturils

Distutils는 패키징이라는게 존재하지 않는 파이썬에서 프로젝트 애플리케이션 구성을 작성한 스크립트를 사용하는 것을 의미하는 것으로 보인다. 설정 스크립트는 파이썬 코드로 작성되며 `setup.py`로 작성한다.

- python 표준 라이브러리에 포함된 기본 패키지 배포 도구
- `setup.py` 스크립트 작성을 통해 패키지 정의
- 기능이 제한적
- `python setup.py install`로 설치

## setuptools

- disturils의 확장된 버전으로 기본 내장은 아니고 서드파티 패키지
- disturils의 모든 기능은 포함하면서 추가 기능 제공
- pip 통합으로 더 편리한 설치

파이썬 3.12 기준으로 distutils가 표준 라이브러리에서 제거되면서 setuptools가 사실상 표준이 됨

## pyproject.toml

- PEP 518에서 정의한 파이썬 프로젝트의 표준 설정 파일
- TOML(Tom's Obvious, Minimal Language) 형식 사용
- 프로젝트 메타데이터, 의존성, 빌드 설정을 선언적으로 정의
- 여러 도구들의 설정을 하나의 파일에 통합 가능

pyproject.toml은 선언형으로 데이터만 명시한다. 그렇기에 단독으로 사용할 수 있는 것은 아니고 setuptool 혹은 여러 빌드 백엔드(poetry, uv, ...)와 함께 사용된다. 그리고 설치해야할 외부 패키지에 대해 버전을 더 명확하게 관리할 수 있고 설정 통합이 가능하다.

그렇기에 현재는 pyproject.toml을 이용한 프로젝트 배포 관리가 권장된다.
