url 인코딩은 공식적으로 `퍼센트 인코딩(persent-encoding)` 이라고 불린다. 주된 목적은 데이터가 URI 내에서 안전하게 사용될 수 있도록 하는 것이다.

좀 더 상세하게는 특수 문자가 URI의 구조와 해석을 방해하지 못하도록 하는 것이다.

- URI 구조의 통일성: URI 내에서는 오직 US-ASCII 문자만 사용하여 임의의 데이터는 인코딩한다.
- 예약된 문자와의 충돌 방지: 슬래시(/) 혹은 URL 구성을 위해 사용되는 예약 문자를 실제 데이터에서 문자로 활용되어야하는 경우 인코딩한다.


## 작동 방식

- 특수 문자 및 예약된 문자의 경우 퍼센트 기호(%) 뒤에 해당 문자의 바이트 값을 나타내는 두 개의 16진수 숫자가 뒤따르는 방식이다. 예를 들어 공백의 경우 `%20` 으로 인코딩된다.
- 퍼센트 기호 자체는 `%25` 로 인코딩된다.
- 바이트 값은 데이터를 1바이트(8비트) 단위로 나누고 각 바이트를 인코딩한다. 예를 들어 바이트 값 `0x0F` 는 `%0F` 로 표현된다.
- non-ascii의 경우 UTF-8 방식으로 바이트 시퀀스로 변환한다. 다음, 각 바이트 값이 위와 동일하게 인코딩된다.
