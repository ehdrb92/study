## HTTP 1.0: 확장성과 메타데이터의 도입

1996년에 발표되었으며. 초기 HTTP/0.9의 제한적인 기능을 확장하여 웹이 다양한 미디어 타입을 처리할 수 있다.

- 비연결성: 가장 큰 특징이다. 브라우저가 각 요청마다 서버와 새로운 연결을 맺고, 요청 처리가 끝나면 연결을 끊어 상태를 저장하지 않는 것이다. 이로 인해 웹 페이지 랜더링을 위한 요청이 많아질 경우 연결을 생성하고 닫는 과정이 빈번하여 성능 저하가 발생할 가능성이 있다.
- 헤더 도입: 요청과 응답에 헤더가 추가되어 메타데이터 전송이 가능해졌다. 이를 통해 단순 텍스트 외에도 이미지, 비디오 등 다양한 `Content-Type` 을 전송 가능해졌다.
- 상태 코드: 성공 혹은 실패 여부를 알 수 있는 상태 코드가 도입되었다.

## HTTP 1.1: 지속적 연결과 표준화

1997년에 발표되어 HTTP/1.0의 성능 문제를 개선하고, 현재까지도 널리 쓰이는 표준이다.

- 지속적 연결: `Connection: keep-alive` 헤더를 통해 한 번 맺은 TCP 연결을 재사용할 수 있게 되어, 매 요청마다 핸드셰이크를 하는 오버헤드를 줄였다.
- 파이프라이닝: 이전 요청의 응답을 기다리지 않고 다음 요청을 연속해서 보낼 수 있는 기능이 도입되었으나, 구현의 어려움과 Head-of-Line Blocking(HOL) 문제로 브라우저 지원이 제한적이었다.
- Host 헤더 지원: HTTP/1.0이 설계될 당시에는 대부분의 웹 서버가 단일 IP 주소로 하나의 웹 사이트만 호스팅하는 것이 일반적이었기 때문에, 클라이언트가 접속할 대상 서버를 구분하기위해 Host 헤더가 필요하지 않았다. 하지만 하나의 IP 주소와 포트를 하진 물리적 서버가 여러 개의 도메인을 서비스할 수 있는 가상 호스팅 기능으로 인해 실제 요청 도메인을 구분하기 위해 Host 헤더를 통한 구분이 필요하여 지원을 시작했다.
- 기능 확장: 캐시 제어 메커니즘이 강화되었고, 파일의 일부분만 요청하는 범위 요청이 가능해져 다운로드 재개 등이 지원되었다. 또한 PUT, DELETE 등의 새로운 메서드가 추가되었다.

## HTTP 2.0: 성능 향상을 위한 이진 프로토콜

2015년에 발표된 구글의 SPDY 프로토콜을 기반으로 만들어진 것으로 성능 효율성에 중점을 두었다.

- 이진 프레이밍: 텍스트 기반인 HTTP/1.x와 달리, 데이터를 바이너리 프레임 단위로 나누어 전송하여 파싱 효율과 전송 속도를 높혔다.
- 멀티플렉싱: 하나의 TCP 연결 내에서 여러 요청과 응답을 동시에 병렬로 주고받을 수 있어, HTTP/1.1 HOL 문제를 해결했다.
- 헤어 압축: 중복되는 헤더 정보를 HPACK 알고리즘을 사용하여 압축함으로서 전송 데이터의 크기를 줄였다.
- 서버 푸시: 클라이언트가 요청하지 않은 리소스를 서버가 미리 보내줄 수 있어 로딩 속도를 새건했다.
- 요청 우선순위: 각 요청에 우선순위를 지정하여 중요한 리소스를 먼저 처리할 수 있다.

## HTTP 3.0: QUIC을 통한 전송 계층의 혁신

TCP 대신 UDP 기반의 QUIC 프로토콜을 사용하여 TCP 고유의 지연 문제를 해결하는게 핵심이다.

- QUCI:  전송 계층에서 TCP 대신 UDP를 사용하는 QUIC을 도입하여 연결 설정 속도를 획기적으로 줄였다. TCP의 3-way handshake 과정을 단축하여 더 빠른 연결이 가능하다.
- TCP의 Head-of-Line Blocking 해결: HTTP/2는 하나의 TCP 연결을 공유하므로 패킷 손실이 발생하면 모든 스트림이 중단되는 문제가 있었으나, HTTP/3(QUIC)는 각 스트림이 독립적이어서 패킷 손실이 다른 스트림에 영향을 주지 않는다.
- QPACK 헤더 압축: QUIC의 특성(패킷 순서가 보장되지 않을 수 있음)에 맞춰 HPACK을 개선한 QPACK 압축 방식을 사용한다.
- 보안 강화: TLS 1.3 암호화가 프로토콜 자체에 내장되어 있어 별도의 설정 없이도 항상 암호화된 통신을 수행한다.
- 연결 마이그레이션: 사용자의 IP가 바뀌어도(예: Wi-Fi에서 LTE로 전환) 연결 ID(Connection ID)를 사용하여 끊김 없이 통신을 유지할 수 있다.
