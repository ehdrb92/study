OpenAI API를 사용하는 중 `data:image/jpeg;base64,{데이터}` 형식으로 base64 데이터를 전달하는 구조를 보았다. 그때는 사실 해당 구조가 OpenAI에서 데이터 타입 확인을 위한 독자적인 방식이라고 생각했다.

하지만 cursor를 이용해 바이브 코딩으로 개발을 하던 중 모델이 위 구조를 사용해서 base64로 인코딩된 데이터를 가져오는 로직을 완성시켰다. 이에 대해 찾아본 결과 해당 구조는 `data uri` 라는 이미 정의된 구조였다.

해당 구조는 컨텐츠 작성자가 작은 크기의 파일을 텍스트로 구성된 문서 내에 인라인으로 삽입하기 위해 탄생했다고 한다.

일반적으로 `data url` 로 더 많이 불린다.

구조는 다음과 같다.

- 접두사: `data:`
- 데이터 타입(MIME): image/jpeg, image/png, ...
- 부가적인 토큰: base64 (해당 데이터가 일반적인 텍스트가 아닌 경우)
- 실제 데이터
- 구문 형식: `data:[<mediatype>][;base64],<data>`
