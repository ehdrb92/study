해시 테이블은 대량의 데이터 볼륨을 가진 경우에도 데이터의 검색, 추가, 삭제를 매우 빠르게 수행할 수 있는 자료 구조이다.

배열 또한 특정 인덱스 접근에 상수 시간 복잡도를 소모하므로 매우 빠르지만, 이는 인덱스로 접근하는 전제 조건이 따른다. 어떠한 의미를 가진 명칭으로 한번에 직접 접근이 가능한 것은 해시 테이블이다.

해시 테이블은 5단계로 구성된다.

1. 빈 배열을 생성한다.
2. 해시 함수를 만든다.
3. 해시 함수를 사용하여 요소를 삽입한다.
4. 해시 함수를 사용하여 요소를 찾는다.
5. 충돌이 발생할 경우 처리한다.

해시 테이블의 사용 사례는 다음과 같다.

- 특정 컬렉션에 무언가 있는지 확인하는 경우 (예시: 도서관의 특정 장르에 책 찾기)
- 고유한 항목을 저장하고 빠르게 찾아야하는 경우 (예시: 주민등록번호, 전화번호)
- 특정 값을 키에 연결하는 경우
