소프트웨어 아키텍처의 주요 특성들은 비기능 요구사항(Non-Functional Requirements)이라고도 불리며, 시스템 성능을 평가하는 데 사용되는 요구사항이다.

## 주요 소프트웨어 아키텍처 특성

- **성능 (Performance)**: 특정 시간 내에 특정 작업을 수행하는 시스템의 반응성을 의미. 응답 시간, 처리량, 자원 활용도 등이 포함된다.
- **가용성 (Availability)**: 시스템이 작동해야 하는 전체 시간 대비 실제 사용 가능한 시간의 비율. 시스템이 필요할 때 얼마나 안정적으로 작동하는지를 나타낸다.
- **신뢰성 (Reliability)**: 시스템이 시간이 지나도 계속 작동을 유지하는 능력으로, 장애 없이 얼마나 오랫동안 안정적으로 동작하는지를 의미한다.
- **확장성 (Scalability)**: 성능 저하 없이 사용량 증가로 인한 스트레스 수요를 처리할 수 있는 시스템의 능력. 사용자나 데이터가 증가해도 시스템이 대응할 수 있는 정도를 나타낸다.
- **보안 (Security)**: 악의적이거나 무단 시도로부터 시스템을 보호하면서 정당한 사용자에게는 접근을 제공하는 능력. 기밀성, 무결성, 인증, 책임추적성 등의 하위 특성이 있다.
- **사용성 (Usability)**: 사용자가 시스템에서 특정 작업을 수행하는 용이성을 의미. 시스템의 조작성, 사용자 오류 방지, UI 심미성 등이 포함된다.
- **유지보수성 (Maintainability)**: 소프트웨어 애플리케이션을 쉽게 유지하고 비용 효율적으로 변경을 지원할 수 있는 능력. 코드 수정, 개선, 버그 수정이 얼마나 쉬운지를 나타낸다.
- **테스트 용이성 (Testability)**: 말씀하신 대로, 사전 정의된 테스트 기준이 충족되었는지 확인하기 위해 테스트를 수행할 수 있도록 시스템이나 컴포넌트가 얼마나 용이하게 하는지를 의미한다.
- **이식성 (Portability)**: 시스템이나 컴포넌트를 한 하드웨어, 소프트웨어 또는 다른 환경으로 전송할 수 있는 정도. 다양한 플랫폼에서 실행 가능한 정도를 나타낸다.
- **상호운용성 (Interoperability)**: 두 개 이상의 소프트웨어 시스템이 데이터를 교환하고 재사용할 수 있는 정도. 외부 시스템과의 통합이 얼마나 쉬운지를 나타낸다.
- **모듈성 (Modularity)**: 말씀하신 대로, 소프트웨어가 개별 구성 요소로 구별되는 정도를 의미. 각 모듈이 독립적으로 개발, 테스트, 수정될 수 있는 정도를 나타낸다.
- **재사용성 (Reusability)**: 기존 소프트웨어를 작거나 변경 없이 다른 소프트웨어에서 사용할 수 있는 능력으로, 비용 효율적이고 시간을 절약할 수 있는 특성이다.
- **복구성 (Recoverability)**: 중단이나 장애 발생 시 소프트웨어 시스템이 영향받은 데이터를 복구할 수 있는 정도이다.

## 추가 참고사항

품질 속성은 기능 요구사항 및 애플리케이션 동작과 긴밀한 관계를 가지며, 여러 전술이 원하는 품질 수준 달성에 도움이 될 수 있지만 불가피하게 다른 속성과 충돌한다. 예를 들어, 보안을 강화하면 성능이 저하될 수 있습니다. 따라서 아키텍트는 이러한 특성들 간의 트레이드오프를 신중히 분석해야 한다.
