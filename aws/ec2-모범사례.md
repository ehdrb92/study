## 보안

- 키 페어를 사용하여 ec2 인스턴스에 접속을 제한
- 보안 그룹(가상 방화벽)을 이용한 인바운드, 아웃바운드 트래픽 제어
  - ssh의 경우 가능한 모든 위치가 아닌 특정 IP로 고정
  - 보안 그룹에 중복되는 규칙이 있다면, 허용 범위가 넓은 규칙이 최종 적용
  - 연결 추적도 가능


## 스토리지

- AWS EBS 볼륨 혹은 AWS S3의 인스턴스 저장소를 루트 볼륨으로 사용가능
  - 다만 인스턴스 저장소는 일부 타입의 인스턴스만 가능하며, 운영체제가 windows이면 불가능
- AWS EBS 볼륨의 경우 인스턴스가 종료되어도 별도로 보존이 가능
  - 다만 보존 시 요금 부과
- 인스턴스의 임시 데이터를 저장하는 데 인스턴스 저장소를 사용가능하다.


## 백업 및 복구

- AWS EBS 스냅샷을 이용하여 정기적으로 백업하고, 인스턴스의 AMI를 만들어 추후 인스턴스 시작을 위한 템플릿을 구성할 수 있다.
- AWS Cloudwatch를 이용하여 EC2 리소스를 모니터링 할 수 있다.
- 인스턴스 및 EBS 볼륨 복구 프로세스를 정기적으로 테스트하여 데이터와 서비스가 정상 복원되는지 확인한다.


## 네트워크

- 애플리케이션의 TTL(Time-to-Live) 값을 IPv4 및 IPv6의 경우 255로 설정한다. 더 작은 값을 사용하면 애플리케이션 트래픽이 전송되는 동안 TTL이 만료되어 인스턴스에 연결성 문제가 발생할 수 있다.
